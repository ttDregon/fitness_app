# Подписки и лимиты — что осталось подключить

Код готов (Фаза 1). Логика: подписки хранятся в Supabase, бэкенд считает лимиты ИИ,
Telegram-бот после оплаты пишет подписку в Supabase, приложение её читает.

## 1) Supabase — выполнить SQL
Supabase → SQL Editor → выполни `backend/subscriptions.sql`
(добавит `profiles.trainer_until/ai_plan/ai_until` и таблицу `ai_usage`).

## 2) Бэкенд — уже задеплоен
Лимиты ИИ (`/chat`, `/parse`, `/parse_meals`) уехали на Render с пушем. Условие:
`SUPABASE_KEY` на Render = **service_role** (уже так). До выполнения SQL бэкенд
не падает — просто не лимитирует, пока нет таблиц.

## 3) Указать своего бота в приложении
`frontend/src/config/billing.ts` → `TG_BOT = 'имя_бота'` (без @), затем выложить по OTA:
`cd frontend && eas update --branch preview -m "set tg bot"`.

## 4) Telegram-бот (создаёшь отдельно) — что он должен делать
Кнопка оплаты в приложении открывает: `https://t.me/<бот>?start=<kind>-<plan>-<userId>`
- `kind` = `trainer` | `ai`
- `plan` (trainer) = `m1|m3|m6|m12` · (ai) = `p50|p150|unlim`
- `userId` = id пользователя в Supabase (auth.users.id)

Бот:
1. Принимает оплату звёздами (Telegram Stars) за нужный тариф.
2. После успешной оплаты пишет в Supabase (под **service_role**):
   - **trainer:** `profiles.trainer_until = now() + N месяцев` (m1→1, m3→3, m6→6, m12→12).
   - **ai:** `profiles.ai_plan = 'p50'|'p150'|'unlim'`, `profiles.ai_until = now() + 1 месяц`.
3. Пользователь возвращается в приложение → «Я оплатил — проверить» подтягивает статус.

Тарифы: тренер 100/250/500/1100 грн (1/3/6/12 мес). ИИ-чат 50→50/день, 150→150/день, 249→безлим.

## 5) Как выдать доступ вручную (для теста/первых продаж до автоматизации бота)
В Supabase → Table editor → `profiles` → нужный пользователь:
- тренер: `trainer_until` = дата в будущем (напр. +1 месяц).
- ИИ: `ai_plan` = `unlim`, `ai_until` = дата в будущем.
Готово — приложение откроет доступ после «проверить» или перезахода.

## 6) Возврат в приложение после оплаты (авто-проверка)
- **Работает сразу (OTA):** когда пользователь жмёт «Оплатить», приложение запоминает
  это и при возврате из Telegram само перепроверяет подписку (несколько попыток) —
  ручное «Я оплатил — проверить» остаётся как запасной вариант.
- **Кнопка «Вернуться в приложение» в боте (после след. пересборки APK):** Telegram
  принимает в кнопках только https, поэтому бот шлёт `https://<backend>/open?kind=trainer|ai`,
  а бэкенд редиректит в схему приложения `mysafeapp://paid?kind=...`. В app.json добавлена
  `"scheme": "mysafeapp"` (нативно регистрируется только при сборке APK, по OTA не активируется).
  Эндпоинт `/open` уже в backend (Render). До пересборки кнопка просто откроет страницу-заглушку.

## Что бесплатно / платно
- 🆓 всем: личный трекинг (вес/вода/питание/журнал), конструктор тренировки, роль клиента.
- 🆓 хук: разбор еды и тренировок — 10/день каждый (всегда), ИИ-чат — 10/день.
- 💎 платно: роль тренера (клубы, назначения, расписание), ИИ-чат сверх 10/день.
